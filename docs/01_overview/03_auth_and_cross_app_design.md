# 認証アプリとクロスアプリ設計

## このドキュメントの目的
- Opinioにおける共通認証とアプリ遷移の前提を明確にする
- ATS単体では完結しない構造を共有する

## 共通認証（Authアプリ）の位置づけ
- Opinioでは、全アプリ共通の認証を専用アプリで行う
- 各業務アプリは**ユーザー認証を行わず**、自アプリ内の認可のみを行う

## 基本的なログインフロー
1. ユーザーは認証アプリにアクセス
2. 認証完了後、利用可能なアプリ一覧が表示される
3. ユーザーはバナー等から利用するアプリを選択
4. 選択したアプリ（例：ATS）へ遷移する

## クロスアプリ遷移の考え方
- 認証状態はアプリ間で共有される
- アプリごとに役割・権限は分離される
- ユーザーは同一ログインで複数アプリを利用可能
- **業務責務と権限判断をアプリ単位で完結させることを目的とする**

## ATS側で意識する責務
- 認証は前提として信頼する
- 自アプリ内の権限・操作範囲のみを判断する
- 他アプリの状態やデータに依存しすぎない

## 将来拡張の前提
- 新しい業務アプリの追加
- アプリ単位での契約・提供範囲の切り替え
- 認証基盤は変更せず拡張可能であること
