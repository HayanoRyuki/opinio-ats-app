# 外部連携チェックリスト

各チャンネルとの連携時に確認・依頼が必要な項目をまとめています。

---

## 1. Web応募（自社採用サイト）

### 連携先
- [ ] 採用サイト制作会社
- [ ] 社内Web開発チーム

### 事前準備
- [ ] company_id を確認（管理画面 → 設定 → 企業情報）
- [ ] 連携対象の求人を作成し、job_id を控える
- [ ] API仕様書を共有（docs/intake-api-guide.md）

### 依頼内容
```
【採用サイト改修依頼】

対象: エントリーフォーム

改修内容:
1. フォーム送信時に以下のAPIへデータを送信
   - URL: POST https://ats.example.com/api/intake/web
   - 形式: JSON

2. 送信データのマッピング
   - company_id: 1（固定）
   - job_id: ページの求人ID
   - name: お名前フィールド
   - email: メールアドレスフィールド
   - phone: 電話番号フィールド（任意）
   - resume_url: 履歴書アップロード後のURL
   - cover_letter: 志望動機フィールド

3. エラーハンドリング
   - success: true → 完了画面表示
   - success: false → エラーメッセージ表示

納期希望: ◯月◯日
```

### 確認事項
- [ ] テスト環境で動作確認
- [ ] 本番環境のURLを最終確認
- [ ] 履歴書ファイルのアップロード先を決定

|-----------|--------|--------|---------|
| 例: リクルートエージェント | 田中様 | tanaka@example.com | ⬜ 未連携 |
| | | | |
| | | | |

### 連携依頼メールテンプレート
```
件名: 採用管理システムとのAPI連携のご相談

◯◯株式会社
◯◯様

お世話になっております。
株式会社△△ 人事部の□□です。

この度、弊社では採用管理システムを刷新いたしました。
候補者様のご推薦時に、以下のAPIをご利用いただくことで
選考状況の共有がよりスムーズになります。

【API連携の概要】
- エンドポイント: POST https://ats.example.com/api/intake/agent
- 送信データ: 候補者情報、推薦コメント等
- 詳細仕様: 添付資料をご参照ください

【メリット】
- 推薦情報の自動登録
- 選考状況のリアルタイム共有（今後対応予定）
- 書類のやり取り削減

ご検討いただけますと幸いです。
ご不明点がございましたら、お気軽にお問い合わせください。
```

### 代替手段（API連携が難しい場合）
- [ ] メールからの手動登録（取り込み管理画面）
- [ ] CSVインポート機能（今後開発予定）

## 3. スカウトサービス

### 連携先リスト
| サービス名 | 契約状況 | 連携方法 | 設定状況 |
|-----------|---------|---------|---------|
| ビズリーチ | ⬜ 契約中 | Webhook / 手動 | ⬜ 未設定 |
| Wantedly | ⬜ 契約中 | API連携 / 手動 | ⬜ 未設定 |
| Green | ⬜ 契約中 | 手動 | - |
| LinkedIn | ⬜ 契約中 | 手動 | - |

### ビズリーチ連携手順
1. [ ] ビズリーチ管理画面にログイン
2. [ ] 設定 → 外部連携 → Webhook設定
3. [ ] URL: `https://ats.example.com/api/intake/scout`
4. [ ] イベント: 応募受付時
5. [ ] テスト送信で動作確認

### Wantedly連携手順
1. [ ] Wantedly Admin にログイン
2. [ ] 設定 → API連携
3. [ ] 応募通知Webhookを設定
4. [ ] テスト送信で動作確認

### 手動運用フロー（API連携できない場合）
```
1. スカウトサービスで応募通知を受信
2. ATS管理画面 → 取り込み管理 → 新規登録
3. チャンネル「スカウト」を選択
4. 候補者情報を入力
5. 登録
```

## 4. リファラル（社員紹介）

### 社内周知
- [ ] 全社メールでリファラル制度を周知
- [ ] 社内ポータルにリファラルフォームを設置
- [ ] 報酬制度を人事規程に明記

### リファラルフォーム設置依頼
```
【社内システム改修依頼】

対象: 社内ポータル

追加機能: リファラル紹介フォーム

要件:
1. ログインユーザーの情報を自動取得
   - 社員番号
   - 氏名
   - メールアドレス
   - 所属部署

2. 入力項目
   - 候補者との関係（選択式: 友人/元同僚/知人/その他）
   - 候補者氏名（必須）
   - 候補者メールアドレス
   - 候補者電話番号
   - 希望ポジション（求人一覧から選択）
   - 推薦コメント

3. API送信
   - URL: POST https://ats.example.com/api/intake/referral
   - 形式: JSON

4. 完了後の表示
   - 「ご紹介ありがとうございました。選考結果は追ってご連絡いたします。」
```

### 運用ルール
- [ ] 紹介者への選考状況フィードバック方法を決定
- [ ] 報酬支払いタイミングを決定（入社時 / 試用期間終了時）
- [ ] 同一候補者の重複紹介時のルールを決定
