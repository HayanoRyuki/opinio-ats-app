# 認証・認可設計（ATS）

## 前提
Opinio ATS は単体アプリではなく、
Opinio 全体のアプリ群の一部として動作する。

## 認証の基本方針
- 認証は app.opinio.co.jp（Authアプリ）が唯一の正
- ATS は独自にログイン機構を持たない
- JWT を用いた SSO により利用者を識別する

## 認可（Authorization）
- JWT 内の company_id / role を基に制御
- 画面・API の両方で権限制御を行う
- UI制御とサーバー制御は必ず二重化する

## 想定ロール
- 管理者
- 採用担当
- 面接官

## 原則
- 権限は最小限
- 不正アクセス時は即拒否
