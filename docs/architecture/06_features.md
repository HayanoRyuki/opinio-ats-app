# 機能群・メニュー

Opinio ATS の機能群は、
「採用判断を再現可能な構造にする」というコンセプトから逆算して設計しています。

私たちは、ATS の機能を
「できることの一覧」ではなく、
「採用における行動の流れ」として捉え直しました。

---

## 機能一覧

### 評価基準
- ポジションごとの評価基準定義
- 面接官への共有
- 評価基準の履歴管理

### 求人管理
- 求人票作成・編集・公開
- 採用ページの作成

### 母集団形成
- 媒体連携（自動取り込み）
- エージェント推薦ページ発行
- リファラル採用ページ
- アンケート機能

### 候補者管理
- 候補者情報の一元管理
- 履歴書／職務経歴書の自動読取・自動転記
- 候補者属性の自動取得
- 重複候補者アラート
- 候補者画面からのワンクリック検索

### 選考管理
- 選考フロー設計
- ステータス管理
- 評価項目カスタマイズ
- 自動書類選考（スコア算出と根拠提示）
- AI面接（必要な候補者にのみ適用）
- 採用判断とその理由の構造化記録

### 日程調整
- カレンダー連携（Google / Outlook）
- 空き枠の自動取得
- Web会議 URL の自動発行
- 候補日回収

### コミュニケーション
- メール送受信
- 自動メール・自動リマインド
- Slack / Chatwork 連携
- タイムラインコメント機能

### エージェント管理
- エージェント別の推薦管理
- 歩留まり分析
- パフォーマンス可視化

### 分析・レポート
- ダッシュボード
- 歩留まり分析
- チャネル別分析
- 面接官別分析
- クロス集計
- 採用判断の振り返り

## 5つの行動モデル

Opinio ATS では、採用における行動を
次の5つに集約しています。

### 1. 決める
**問い：** 何を基準に採用するのか  
**対応機能：**
- 評価基準
- 求人管理

### 2. 集める
**問い：** どこから候補者を集めるか  
**対応機能：**
- 母集団形成
- エージェント管理

### 3. 見極める
**問い：** この人は基準に合っているか  
**対応機能：**
- 候補者管理
- 選考管理
- 日程調整

### 4. 残す
**問い：** なぜその判断をしたのか  
**対応機能：**
- 選考管理（判断理由の記録）
- コミュニケーション

### 5. 学ぶ
**問い：** 次の採用に活かせるか  
**対応機能：**
- 分析・レポート

## なぜこの機能構成なのか

従来の ATS が効率化してきたのは、
「集める → 見極める」の部分だけでした。

しかし、採用の失敗は、
その前の「決める」と、
その後の「残す・学ぶ」で起きています。

Opinio ATS は、
この抜け落ちていた3つの行動を補うことで、
採用を一時的な作業ではなく、
組織に蓄積される意思決定プロセスへと変えます。

### 採用判断の記録

Opinio ATS では、選考の進行状況（Application.status）と、
採用可否の判断（HiringDecision）を分離して管理する。

これにより、
「なぜその判断に至ったのか」
「判断後の運用は適切だったか」
を後から検証できる構造としている。
