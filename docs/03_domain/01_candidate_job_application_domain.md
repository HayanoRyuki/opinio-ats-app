# 候補者・求人・応募 責務領域

## 責務領域の位置づけ
本責務領域は、Opinio ATS における  
**採用活動の事実の正（Single Source of Truth）** を扱う。

ここで扱うのは、
- 誰（候補者）が存在するか
- どの求人が存在するか
- 両者がどのような関係（応募）を持つか

という **確定した事実情報のみ** である。

本責務領域は、  
評価・判断・進行の結論を一切持たない。  
それらは明確に **他の責務領域の責任範囲** とする。

---

## 扱う対象（中核概念）
- Person  
  （Opinio 全体で共通となる人物の概念）
- Candidate  
  （ATS 上で採用判断の対象となりうる人物）
- Job  
  （募集・検討対象となる求人）
- Application  
  （Candidate と Job の間に成立する「応募」という関係）

---

## 設計上の前提
- 候補者（Candidate）は応募前から存在しうる
- 1人の候補者が複数求人に応募できる
- 応募（Application）は「関係」であり、候補者の属性ではない
- 原データは原則変更せず、保持を優先する
- 状態の変化は「上書き」ではなく「履歴」として扱う前提とする
- **本責務領域に存在する Application は、すべて「確定した応募」である**

---

## 応募（Application）の成立条件（重要）
Application は、以下の条件を満たした場合にのみ  
本責務領域に生成される。

- 応募チャネル責務領域において
  - 通知・入力内容が整理され
  - 人による確認・補正を経て
  - 「正式応募として扱う」と判断されたもの

ダイレクトリクルーティング等における  
**仮応募・反応段階のデータは、本責務領域には存在しない。**

それらは応募チャネル責務領域の責任範囲とする。

---

## この責務領域でやること
- 候補者の存在管理  
  （Candidate の生成・参照）
- 求人情報の管理  
  （Job の生成・参照）
- 応募という関係の生成・保持  
  （Application の生成・参照）
- 応募に関する **事実としての状態管理**  
  （例：応募済／取り下げ／終了 など）

※ 進行状況・評価状態・判断理由は扱わない

---

## この責務領域でやらないこと
- 選考ステップの定義・管理
- 面接や選考進行の判断
- 評価・所感・解釈の付与
- 採否・処遇の確定
- 点数化・ランク化・優劣付け
- 仮応募・未確定応募の保持

本責務領域は、  
**判断の材料となる「確定した事実」を正しく保持することにのみ責任を持つ。**

---

## 他責務領域との関係
- 選考パイプライン責務領域  
  → Application を参照し、進行状態（ステップ）を管理する
- 評価・判断補助責務領域  
  → Application / Candidate を参照し、  
    思考材料（観点・質問・メモ）を紐づける
- 応募チャネル責務領域  
  → 通知・入力を整理し、  
    **確定した Candidate / Application のみを本責務領域へ引き渡す**

本責務領域は、  
他責務領域の判断や意味づけを代替しない。

---

## 一文定義（迷ったとき用）
候補者・求人・応募 責務領域は、  
**採用に関わる「確定した事実」を正として保持するための責務領域である。**
